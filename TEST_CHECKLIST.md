### 3. 自动重试机制

#### 测试场景 1：网络故障自动重试
- [ ] 打开浏览器开发者工具（F12）→ Network 标签
- [ ] 上传音频文件，准备转写
- [ ] 点击"开始转写"，立即在 Network 标签中找到请求
- [ ] **右键请求 → Block request URL**（模拟网络故障）
- [ ] 观察 Console，应出现 `[ASR Retry] 第 1 次重试` 日志
- [ ] 等待几秒，出现 `[ASR Retry] 第 2 次重试`
- [ ] 取消阻止（Unblock），第 3 次重试成功

**预期结果：**  
✅ 最多重试 3 次  
✅ 延迟递增（1s → 2s → 4s）  
✅ 最终成功或显示错误提示

#### 测试场景 2：后端服务停止
- [ ] 停止后端 ASR 服务（杀进程或关闭）
- [ ] 上传音频，开始转写
- [ ] 观察 Console，出现 3 次重试日志
- [ ] 最终 Toast 提示"无法连接到 ASR 服务，请确认后端已启动"

**预期结果：**  
✅ 重试 3 次后仍失败，显示友好错误提示  
✅ 不会无限重试卡死界面

#### 测试场景 3：用户取消不重试
- [ ] 上传音频，开始转写
- [ ] 在转写过程中点击"重置"按钮（取消）
- [ ] 观察 Console，**不应出现**重试日志
- [ ] Toast 提示"转写已取消"

**预期结果：**  
✅ 用户主动取消不触发重试

## 🐛 边界情况测试

### 批量导出
- [ ] 队列中全部文件失败 → Toast 提示"没有可导出的文件"
- [ ] 导出过程中关闭页面 → 重新打开，ZIP 已下载（浏览器下载管理器）
- [ ] 文件名包含特殊字符（如 `音频/测试.mp3`） → 导出后文件名为 `音频_测试.txt`

### 音频预处理
- [ ] 上传损坏的音频文件 → Toast 提示"音频解码失败"，不进入转写
- [ ] 预处理超长音频（>10分钟） → 预处理时间较长，但最终成功
- [ ] 预处理已经是 16kHz WAV → 跳过重采样，仅做归一化

### 自动重试
- [ ] 后端返回 400 错误（客户端错误） → 不重试，直接显示错误
- [ ] 后端返回 500 错误（服务器错误） → 重试 3 次
- [ ] 转写超时（>2分钟无响应） → 重试机制生效

### IndexedDB
- [ ] 浏览器隐私模式 → IndexedDB 可能不可用，自动降级为内存存储
- [ ] 浏览器配额已满 → Console 警告，新记录无法保存
- [ ] 手动删除 IndexedDB 数据库 → 刷新后重新初始化，历史清空
