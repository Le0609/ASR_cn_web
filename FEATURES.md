# 新功能使用指南

本次更新为 Echo 语音转文字应用新增了四项核心功能，提升了多文件处理效率、系统稳定性和识别准确率。

---

## ✨ 新增功能概览

### 1. 批量导出（ZIP 压缩包）

**功能说明：**  
当使用多文件队列转写时，可一键导出所有已完成文件为单个 ZIP 压缩包，避免逐个下载。

**使用方法：**
1. 上传多个音频文件（2个或以上）
2. 选择转写模式并完成转写
3. 在"导出结果"区域会出现 **"批量导出全部 (N 个文件)"** 按钮
4. 点击按钮，自动生成 ZIP 文件并下载

**技术细节：**
- 使用 JSZip 库打包（已内置）
- 文件名格式：`转写结果_20260709_153045.zip`
- 压缩包内结构：`transcripts/文件名1.txt`、`transcripts/文件名2.txt`...
- 支持进度展示（打包过程可见）

---

### 2. 音频预处理

**功能说明：**  
在转写前自动优化音频质量，包括音量归一化、降噪、格式统一转换，提升 ASR 识别准确率。

**使用方法：**
1. 上传音频文件后，在"选择转写模式"面板中
2. 找到 **"音频预处理"** 开关（默认开启）
3. 开启后会自动执行以下优化：
   - 重采样到 16kHz（ASR 标准采样率）
   - 转为单声道（减少干扰）
   - 音量归一化（峰值 95%，避免过载或过小）
   - 简单降噪（去除低频环境噪声）

**适用场景：**
- 手机录音（采样率不统一）
- 环境噪声较大的音频
- MP3/M4A 等非 WAV 格式

**性能影响：**
- 增加 2-5 秒预处理时间（视文件大小）
- 进度条前 20% 为预处理，后 80% 为转写

**关闭预处理：**  
如果音频已经是高质量的 16kHz WAV，可关闭预处理节省时间。

---

### 3. 转写失败自动重试

**功能说明：**  
网络不稳定或后端临时故障时，自动重试转写请求（最多 3 次），无需手动操作。

**重试策略：**
- 最大重试次数：3 次
- 延迟策略：指数退避（1秒 → 2秒 → 4秒）
- 智能判断：
  - ✅ 网络错误、超时 → 自动重试
  - ✅ 服务器 5xx 错误 → 自动重试
  - ❌ 用户取消 → 不重试
  - ❌ 客户端错误（文件格式、大小） → 不重试

**用户体验：**
- 重试过程静默进行，不打断操作
- 开发模式下可在控制台看到 `[ASR Retry] 第 N 次重试` 日志
- 最终失败时显示错误提示

**适用场景：**
- 移动网络不稳定
- 后端服务重启中
- 偶发性网络波动

---

### 4. IndexedDB 数据持久化

**功能说明：**  
历史记录从 localStorage（5-10MB 限制）升级到 IndexedDB（无限容量），支持存储更多历史和完整音频。

**核心改进：**
- **容量提升：** localStorage ~5MB → IndexedDB 理论无限（实际受浏览器配额限制，通常 >50MB）
- **历史上限：** 20 条 → 50 条
- **音频存储：** 完整保留音频文件（base64 编码），历史记录可完整回放
- **自动迁移：** 首次启动自动从 localStorage 迁移旧数据

**使用方法：**
无需手动操作，升级后自动生效。

**数据迁移：**
1. 首次打开应用时，自动检测 localStorage 中的旧数据
2. 批量迁移到 IndexedDB
3. 迁移成功后清理 localStorage
4. 控制台输出 `Migrated N records from localStorage to IndexedDB`

**兼容性：**
- 支持所有现代浏览器（Chrome、Firefox、Safari、Edge）
- 不支持 IndexedDB 的浏览器自动降级为内存存储

**数据安全：**
- 所有数据仅存储在本地浏览器，不上传服务器
- 清除浏览器数据会同时删除 IndexedDB 记录

## 🐛 故障排查

### 批量导出失败

**症状：** 点击按钮无反应或下载失败  
**原因：** 队列中无已完成文件或浏览器阻止下载  
**解决：**
1. 确认至少有 2 个文件转写完成
2. 检查浏览器是否拦截了弹出下载
3. 查看控制台错误日志

### 音频预处理卡死

**症状：** 进度条停在前 20%  
**原因：** 音频文件损坏或格式不兼容  
**解决：**
1. 关闭预处理开关重试
2. 使用音频编辑软件（如 Audacity）重新导出为标准 WAV
3. 检查文件是否真的是音频（MIME 类型伪装）

### 重试多次仍失败

**症状：** Toast 提示"转写出错"，已重试 3 次  
**原因：** 后端服务未启动或端口错误  
**解决：**
1. 确认后端服务运行：`curl http://localhost:8000/health`
2. 检查控制台 Network 面板查看请求状态
3. 查看后端日志排查具体错误

### IndexedDB 迁移失败

**症状：** 控制台报错 `Failed to migrate from localStorage`  
**原因：** 浏览器 IndexedDB 被禁用或配额已满  
**解决：**
1. 检查浏览器设置，允许站点存储数据
2. 清理浏览器缓存释放空间
3. 使用隐私模式测试（可能是扩展干扰）

## 🎯 最佳实践

1. **批量转写场景：**  
   上传 5-10 个文件 → 开启预处理 → 使用精准模式 → 批量导出 ZIP

2. **单文件高质量转写：**  
   上传高质量 WAV → 关闭预处理 → 使用精准模式 → 导出单个 TXT

3. **移动网络转写：**  
   开启预处理 → 使用快速模式（减少传输时间）→ 依赖自动重试

4. **历史记录管理：**  
   定期清理不需要的历史（右侧面板"清空"按钮）→ 释放浏览器存储空间
